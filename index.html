31/08/2024 . Las Compuertas Lógicas: Los Ladrillos de la Computación

Las Compuertas Lógicas son circuitos electrónicos conformados internamente por transistores que se encuentran con arreglos especiales con los que otorgan señales de voltaje como resultado o una salida de forma booleana, están obtenidos por operaciones lógicas binarias (suma, multiplicación). También niegan, afirman, incluyen o excluyen según sus propiedades lógicas. Estas compuertas se pueden aplicar en otras áreas de la ciencia como mecánica, hidráulica o neumática.

Existen diferentes tipos de compuertas y algunas de estas son más complejas, con la posibilidad de ser simuladas por compuertas más sencillas. Todas estas tienen tablas de verdad que explican los comportamientos en los resultados que otorga, dependiendo del valor booleano que tenga en cada una de sus entradas.



Trabajan en dos estado, "1" o "0", los cuales pueden asignarse a la lógica positiva o lógica negativa. El estado 1 tiene un valor de 5v como máximo y el estado 0 tiene un valor de 0v como mínimo y existiendo un umbral entre estos dos estados donde el resultado puede variar sin saber con exactitud la salida que nos entregara. Las lógicas se explican a continuación:
  • La lógica positiva es aquella que con una señal en alto se acciona, representando un 1 binario y con una señal en bajo se desactiva. representado un 0 binario.
  • La lógica negativa proporciona los resultados inversamente, una señal en alto se representa con un 0 binario y una señal en bajo se representa con un 1 binario.

A continuación vamos a analizar algunas operaciones lógicas una por una de las 3 principales:

Compuerta lógica AND:

También puede definirse como una multiplicación Booleana: Si el valor de todas las variables de entrada es 1, entonces el resultado en la salida será 1 lógico, si por el contrario alguna de las variables de entrada es igual a 0, la salida valdrá 0 lógico.



Compuerta lógica OR:

También definida como una suma Booleana: Siempre que, al menos una de las entradas tenga un valor igual a 1, la compuerta OR dará como resultado un 1 lógico, pero si todas las variables de entrada tienen el valor 0, la salida será un 0 lógico.



Compuerta lógica NOT:

También definida como negación Booleana: Cualquiera que sea el valor en la entrada de la compuerta, 1 o 0, la salida será lo contrario a esta. Cabe mencionar que solo es posible tener una entrada.



26/09/2024 . Tutorial manejo basico de git

Git es una herramienta fundamental para desarrolladores de software que nos permite gestionar los cambios en nuestro código de manera eficiente. En este tutorial, te guiaré paso a paso a través de los conceptos básicos de Git, desde la creación de un repositorio local hasta la conexión con un repositorio remoto en GitHub. luego de descargar git desde su pagina oficial y realizar su instalacion (solo abre el archivo descargado y dale siguiente, siguiente siguiente y asi como cualquier instalacion)

1. ubicacion o creacion de carpeta del proyecto

Primero debes crear una carpeta para tu nuevo proyecto o ubicar un proyecto ya creado

Para saber en que carpeta te encuntras utiliza el comando "ls"
y para moverte por las carpetas utiliza el comando "cd" + el nombre de la carpeta



2. Crear el repositorio de git

ahora para crear el repositorio git en nuestro proyecto debes utilizar el comando "git init"



Este comando crea un nuevo directorio oculto llamado .git que contiene toda la información de tu repositorio y te permitira gestionar las versiones y modificaciones de estas.



3. Creación de ramas (branches)

Las ramas son copias independientes de tu proyecto. Esto te permite trabajar en diferentes funcionalidades sin afectar la versión principal, para reaizar la copia utiliza el comando "git branch + el nombre que desees".



como puedes ver el comando tiene un agregagdo que es el "-b" con esto ademas de crear la rama, te mueve directamente a tu nueva rama creada pero supongamos que solo creaste la rama. Para cambiar a nuestra nueva rama en Git o a cualquier otra rama que hayamos creado anteriormente. podemos hacer uso del comando "git checkout + el nombre de la rama".


¿Cuándo usar ramas?

  • Desarrollar nuevas funcionalidades: Crea una nueva rama para cada característica, evitando conflictos con el código principal.
  • Experimentar: Prueba nuevas ideas sin afectar el código estable.
  • Corregir errores: Crea una rama para solucionar un bug específico.

4. Realizar cambios a nuestras ramas (branch)

ahora puedes abrir tu archivo de proyecto con el comando "code index.html" (debes tener Vscode instalado) y puedes comenzar a trabajar en tu proyecto y git identificara los cambios que realices en la rama como se puede ver en el ejemplo en este caso en la rama "version-editable".



como se ve en la imagen realice cambios a mi documento y utilice el comando "git status" para confirmar que se han realizado cambios en mi proyecto

ahora, cuando termines de trabajar en tu proyecto podemos agregar las modificaciones con el comando "git add index.html
